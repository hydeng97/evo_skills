# 一、文档目的

本文档用于定义 `evo_skills` 在蒸馏开始前的 **材料类型判定规则** 与 **目标产物判定规则**。

其核心目标是避免以下错误：

- 把解释型文章硬写成操作型 skill
- 把参考文档误当教程
- 把会议纪要误做成通用 skill

因此，这份规则应作为 distillation workflow 的第一道门。

# 二、第一层判断：材料类型

## 1. `article`

特征：

- 以观点、论证、案例为主
- 经常有核心 thesis
- 操作步骤不一定完整

蒸馏重点：

- thesis
- 规则
- 适用边界
- 常见误用

默认目标：

- 原则型 skill
- 教练型 skill

默认 execute：

- 关闭

## 2. `book`

特征：

- 内容较长，按章节组织
- 模型、框架、案例反复出现
- 逻辑结构通常比文章更厚重

蒸馏重点：

- 章节地图
- 核心模型
- 跨章节重复原则
- 反模式

默认目标：

- 框架型 skill
- 教练型 skill
- 反思型 skill

默认 execute：

- 关闭

## 3. `tutorial`

特征：

- 以步骤、环境、操作顺序为主
- 有明确输入输出
- 有成功与失败信号

蒸馏重点：

- 前置条件
- 步骤
- 成功判据
- 排错分支
- 回滚方式

默认目标：

- 操作型 skill

默认 execute：

- 候选开启

## 4. `reference`

特征：

- 以参数、接口、限制、能力边界为主
- 完整但往往不直接面向任务
- 常见于 API 文档、官方说明、规范手册

蒸馏重点：

- 能力边界
- 参数与限制
- 版本差异
- 常用任务包装层

默认目标：

- 参考型 skill
- 混合型 skill

默认 execute：

- 谨慎，仅在任务包装明确后评估

## 5. `notes`

特征：

- 片段化
- 上下文残留多
- 重复与冗余明显

蒸馏重点：

- 聚类
- 去重
- 重复出现的判断标准

默认目标：

- 轻量 skill
- memory

默认 execute：

- 关闭

## 6. `transcript`

特征：

- 口语化
- 结构松散
- 常混合事实、意见、未定讨论

蒸馏重点：

- 决策
- 约束
- 禁忌
- 优先级

默认目标：

- memory
- guardrail
- 项目约束型 skill

默认 execute：

- 关闭

# 三、第二层判断：目标产物类型

## 1. 原则型 skill

适合：

- 文章
- 博客
- 书籍中的思想部分

核心输出：

- 核心原则
- 适用场景
- 边界
- 反例

## 2. 教练型 skill

适合：

- 成长型文章
- 决策类材料
- 书籍框架

核心输出：

- 概念解释
- 情景迁移
- 提问引导
- 长期反思支持

## 3. 操作型 skill

适合：

- 教程
- 操作指南
- 标准工作流

核心输出：

- 前置条件
- 步骤
- 验证方式
- 回滚方式

## 4. 参考型 skill

适合：

- 官方文档
- API 文档
- 规范与手册

核心输出：

- 能力边界
- 参数说明
- 常用任务包装层

## 5. memory / guardrail

适合：

- 会议纪要
- 项目约束
- 转录材料
- 临时但重要的决策背景

核心输出：

- 已决定事项
- 约束
- 禁止事项
- 后续执行守则

# 四、判定流程

推荐按以下顺序判定：

## 第一步：先问“这份材料主要是在解释、指导、参考，还是记录？”

- 解释为主 -> `article` / `book`
- 指导为主 -> `tutorial`
- 参考为主 -> `reference`
- 记录为主 -> `notes` / `transcript`

## 第二步：再问“这份材料最终最适合让 agent 做什么？”

- 解释与启发 -> 原则型 / 教练型 skill
- 操作与执行 -> 操作型 skill
- 查阅与约束 -> 参考型 skill / memory

## 第三步：最后问“它是否值得 build 成正式 child skill？”

如果答案是：

- 可重复复用
- 边界明确
- 触发清晰
- 结构稳定

则可以进入正式 skill 流程。

否则，应先保留在 article package 或 memory 层。

# 五、是否适合开启 execute 的规则

## 默认允许作为 execute 候选的材料

- `tutorial`
- 一部分任务包装清晰的 `reference`

## 默认不建议开启 execute 的材料

- `article`
- `book`
- `notes`
- `transcript`

## 例外情况

如果某篇文章或某本书章节本身就是明确步骤型内容，也可以进入 execute 候选，但必须说明：

- 为什么它不是普通 explanation
- 为什么它具备重复、稳定、可验证的步骤结构

# 六、失败信号

如果出现下面这些信号，说明类型判定有问题：

- 文章被强行写成长步骤列表
- 教程完全没有前置条件与验证方式
- 官方参考文档被压缩得只剩口号
- 会议纪要被写成“通用最佳实践”

出现这些情况时，应回到材料类型判定步骤重新分类。

# 七、结论

对于 `evo_skills`，类型判定不是可选增强，而是蒸馏质量的前置门槛。

后续任何自动化或半自动化流程，都应默认先经过本文档定义的两层判断：

1. 材料类型判断
2. 目标产物类型判断
